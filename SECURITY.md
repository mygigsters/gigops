# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in GigOps, please report it responsibly.

**Email:** security@mygigsters.com.au

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge your report within 48 hours and aim to release a fix within 7 days for critical issues.

## Scope

GigOps runs locally on your machine. Security concerns primarily involve:
- API key handling and storage
- Data privacy (benchmark data, profile information)
- Dependency vulnerabilities

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | ✅        |

## Disclosure Policy

We follow coordinated disclosure. Please do not open public issues for security vulnerabilities.
